Top 50 Java Q&A

Key questions covering Units I-IV

Unit 1: Java Basics & How it Works (JVM)

Q: What is Java in simple terms?

  • Widely used, object-oriented programming language.
  • Designed for platform independence ("Write Once, Run Anywhere").
  • Used for web apps, mobile apps, enterprise systems, etc.

Q: What are some main features of Java?

  • Object-Oriented: Organizes code around objects (data + behavior).
  • Platform-Independent (WORA): Runs on any machine with a JVM via bytecode.
  • Simple: Relatively easy to learn (no explicit pointers, etc.).
  • Secure: Features like bytecode verification, Security Manager.
  • Robust: Strong type-checking, exception handling, garbage collection.
  • Multithreaded: Supports concurrent execution.
  • High Performance: Uses JIT compilation for speed.

Q: What does "Write Once, Run Anywhere" mean in Java?

  • Compile Java code once into platform-neutral bytecode (.class files).
  • Run the same bytecode on any device with a compatible JVM.
  • JVM translates bytecode to native machine code for the specific platform.

Q: How does a Java program run (simple steps)?

  • Write: Create .java source code files.
  • Compile: Use `javac` to create .class bytecode files.
  • Load: Use `java` to start JVM; JVM's Class Loader loads .class files.
  • Verify: Bytecode Verifier checks code safety and correctness.
  • Execute: Execution Engine interprets bytecode and/or uses JIT compilation to run native code.

Q: What does the `javac` command do?

  • It's the Java Compiler (part of JDK).
  • Reads .java source files.
  • Checks for syntax and type errors.
  • Translates source code into Java bytecode (.class files).

Q: What does the `java` command do?

  • It's the Java Application Launcher (part of JRE/JDK).
  • Starts the Java Virtual Machine (JVM).
  • Loads the specified main class.
  • Begins execution by calling the `main` method.

Q: What is bytecode?

  • Low-level, platform-independent instruction set.
  • Generated by the Java compiler (`javac`).
  • Executed by the Java Virtual Machine (JVM).
  • Enables "Write Once, Run Anywhere".

Q: What is the JVM (Java Virtual Machine)?

  • An abstract machine specification and a concrete software implementation.
  • Provides the runtime environment for executing Java bytecode.
  • Handles class loading, verification, memory management (GC), and execution.
  • Makes Java platform-independent.

Q: What's the difference between JDK, JRE, and JVM?

  • JVM: Executes bytecode (the "engine").
  • JRE: JVM + Core Java Libraries (needed to *run* Java apps).
  • JDK: JRE + Development Tools (compiler `javac`, debugger, etc.) (needed to *develop* Java apps).
  • Hierarchy: JDK contains JRE, which contains JVM.

Q: What does the Bytecode Verifier do?

  • Security component within the JVM.
  • Checks loaded bytecode *before* execution.
  • Ensures code adheres to JVM specs, is safe, and won't corrupt the JVM (checks types, stack, access rules).

Q: Where does the JVM store things in memory?

  • Heap: Stores all objects and arrays (shared among threads, managed by GC).
  • Java Stack: Stores method call frames (local variables, operand stack) for each thread (thread-private).
  • Method Area: Stores class-level data (bytecode, static variables, runtime constants) (shared among threads).
  • PC Registers: Stores current instruction address for each thread (thread-private).
  • Native Method Stacks: Used for non-Java (native) code execution (thread-private).

Q: What is Garbage Collection (GC)?

  • Automatic memory management process in the JVM.
  • Identifies and reclaims memory in the Heap used by objects that are no longer referenced.
  • Prevents memory leaks and eliminates the need for manual memory deallocation.

Q: What is the Java Sandbox?

  • A restricted execution environment for untrusted code (e.g., old applets).
  • Limits code's access to system resources (files, network) based on a security policy.
  • Enforced by the Security Manager.
  • Protects the host system from malicious code.

Unit 2: Java Programming Basics

Q: What are primitive data types in Java?

  • Basic, non-object data types built into Java.
  • Hold actual values directly.
  • Eight types: `byte`, `short`, `int`, `long`, `float`, `double`, `char`, `boolean`.

Q: What's the difference between primitive and reference types?

  • Primitives: Store actual value; fixed size; live on stack (if local); cannot be `null`.
  • References: Store memory address (pointer) to an object on the Heap; object size varies; can be `null`.
  • Examples: Primitives (`int`, `boolean`); References (`String`, `ArrayList`, arrays, any class).

Q: What are Wrapper Classes? (e.g., `Integer`)

  • Classes that wrap primitive values into objects (e.g., `Integer` for `int`).
  • Needed for Collections (which store objects), utility methods, generics, allowing `null`.
  • Support autoboxing (primitive -> wrapper) and unboxing (wrapper -> primitive).

Q: How do you create an array?

  • Declare: `int[] scores;` (Declare a reference variable).
  • Instantiate: `scores = new int[10];` (Create the array object of a fixed size in memory).
  • Initialize (Optional): `int[] scores = {95, 88, 72};` (Combine declaration, instantiation, initialization).
  • Access elements using zero-based index: `scores[0]`.

Q: What's the difference between `==` and `.equals()` for objects?

  • `==`: Compares memory addresses (reference equality). Checks if two variables point to the *exact same object instance*.
  • `.equals()`: Compares object content/state (value equality). Checks if two objects are meaningfully equivalent. (Must be overridden correctly, e.g., in `String`).
  • Guideline: Use `.equals()` for object content comparison (esp. Strings), `==` for primitives or exact instance check.

Q: What do `if-else` and `switch` do?

  • `if-else`: Executes code blocks based on boolean conditions (`if`, `else if`, `else`). Handles complex conditions.
  • `switch`: Selects a code block based on the value of an expression (`byte`, `short`, `char`, `int`, `String`, `enum`). Uses `case` labels, `break` (to prevent fall-through), and optional `default`. Often clearer for multiple discrete values.

Q: What do `while` and `for` loops do?

  • `while`: Repeats code while a condition is true (checked *before* each iteration).
  • `do-while`: Repeats code while a condition is true (checked *after* each iteration; runs at least once).
  • `for`: Structured loop with initialization, condition, and update parts (often used for counter-based iteration).
  • Enhanced `for` (for-each): Simple syntax to iterate over elements in arrays or collections.

Q: What is a Class? What is an Object?

  • Class: A blueprint or template defining properties (fields) and behaviors (methods) for objects.
  • Object: An instance of a class; a concrete entity created from the blueprint, existing in memory (Heap), with its own state.

Q: What do `public`, `private`, etc. (access modifiers) mean?

  • Control visibility/accessibility:
  • `public`: Accessible from anywhere.
  • `private`: Accessible only within the declaring class.
  • `protected`: Accessible within the package and by subclasses (even in different packages).
  • `default` (package-private): Accessible only within the same package.

Q: What does the `static` keyword do?

  • Indicates a member belongs to the *class itself*, not individual objects.
  • Static Variable: One copy shared by all instances. Accessed via `ClassName.variableName`.
  • Static Method: Called via `ClassName.methodName()`. Cannot directly access instance members (`this`). Used for utilities, `main` method.

Q: What is an Interface?

  • A reference type defining a contract of abstract methods (and constants, default/static methods).
  • Cannot be instantiated directly.
  • A class `implements` an interface, promising to provide code for its abstract methods.
  • Supports multiple inheritance of type (a class can implement many interfaces).
  • Used for polymorphism and defining capabilities.

Q: What is an Abstract Class?

  • A class declared `abstract`; cannot be instantiated directly.
  • Designed to be extended (subclassed).
  • Can contain both `abstract` methods (no implementation) and concrete methods (with implementation).
  • Subclasses must implement inherited abstract methods (unless they are also abstract).
  • Can have fields and constructors.
  • A class can `extend` only one abstract (or concrete) class.

Q: What is Inheritance? (`extends`)

  • Mechanism where a subclass acquires fields and methods from a superclass.
  • Uses the `extends` keyword.
  • Establishes an "is-a" relationship.
  • Promotes code reuse and polymorphism.
  • Java supports single class inheritance, multiple interface implementation.

Q: What is Method Overriding?

  • A subclass provides its own implementation for a method inherited from its superclass.
  • Must have the same signature (name, parameters) and same/covariant return type.
  • Access level cannot be more restrictive.
  • `final` methods cannot be overridden.
  • Use `@Override` annotation for safety.
  • Enables runtime polymorphism.

Q: What is Method Overloading?

  • Defining multiple methods with the *same name* but *different parameter lists* within the *same class*.
  • Difference can be number, type, or order of parameters.
  • Return type alone is not enough.
  • Resolved at compile time (static polymorphism).
  • Used for providing variations of a method.

Q: What is an Exception?

  • An event during program execution that disrupts the normal flow (e.g., `ArithmeticException`, `NullPointerException`, `FileNotFoundException`).
  • Represented by objects inheriting from `Throwable`.
  • Signals an error or unexpected condition.

Q: How do you handle exceptions? (`try`, `catch`, `finally`)

  • `try`: Enclose code that might throw an exception.
  • `catch`: Handle specific exception types thrown in `try`. Executes if a matching exception occurs.
  • `finally`: Optional block guaranteed to execute after `try` (and any `catch`), regardless of exceptions. Used for cleanup (e.g., closing resources).

Q: What's the difference between `String` and `StringBuffer`/`StringBuilder`?

  • `String`: Immutable (cannot be changed after creation). Operations create new `String` objects. Thread-safe.
  • `StringBuffer`: Mutable (can be changed). Operations modify the object in place. Thread-safe (synchronized methods). Slower due to synchronization.
  • `StringBuilder`: Mutable. Operations modify in place. **Not** thread-safe. Faster than `StringBuffer`.
  • Use: `String` for constants, `StringBuilder` for single-threaded modifications, `StringBuffer` for multi-threaded modifications.

Unit 3: Doing Multiple Things at Once (Threads) & GUIs (AWT)

Q: What is a Thread?

  • Smallest unit of execution within a process.
  • Allows concurrent execution of tasks within a program.
  • Threads in the same process share memory (Heap, Method Area).
  • Used for responsiveness, concurrency, parallelism.

Q: How can you create a Thread? (Two ways)

  • Extend `Thread` Class: Subclass `Thread`, override `run()`. Create instance, call `start()`. (Limits further inheritance).
  • Implement `Runnable` Interface: Implement `Runnable`, provide `run()`. Create `Runnable` instance, pass to `new Thread()`, call `start()` on `Thread` object. (Preferred way - more flexible).

Q: How do you start a Thread?

  • Call the `start()` method on the `Thread` object.
  • `start()` allocates resources, schedules the thread, and invokes its `run()` method in a new thread of execution.
  • Never call `run()` directly; it executes in the current thread, not concurrently.

Q: What is Thread Synchronization? (`synchronized`)

  • Mechanism to control access to shared resources by multiple threads.
  • Prevents race conditions and data corruption.
  • Uses the `synchronized` keyword:
  • Synchronized Method: Locks the object (`this`) before execution.
  • Synchronized Block: Locks a specified object before executing the block.
  • Ensures only one thread executes the critical section at a time for the locked object.

Q: What do `wait()`, `notify()`, `notifyAll()` do?

  • Methods of `java.lang.Object` for inter-thread communication.
  • Must be called within a `synchronized` block/method on the object being used for coordination.
  • `wait()`: Releases lock, thread waits until notified (or timeout/interrupted). Must re-acquire lock to proceed. Usually used in a loop checking a condition.
  • `notify()`: Wakes up *one* arbitrary waiting thread on the object's monitor.
  • `notifyAll()`: Wakes up *all* waiting threads on the object's monitor. (Generally safer).

Q: What is AWT?

  • Abstract Window Toolkit - Java's original GUI framework.
  • Platform-dependent: Uses native OS components ("peers"). Look and feel varies across OS.
  • Provides basic components (`Button`, `Frame`, etc.), layout managers, event handling.
  • Largely superseded by Swing and JavaFX but foundational.

Q: What's a Layout Manager in AWT?

  • Object responsible for positioning and sizing components within a container.
  • Handles GUI arrangement automatically, adapting to resizing and platform differences.
  • Examples: `FlowLayout` (rows), `BorderLayout` (regions), `GridLayout` (grid), `GridBagLayout` (flexible grid).
  • Set on a container using `setLayout()`.

Q: How does event handling work in AWT? (Source, Event, Listener)

  • Uses the Event Delegation Model:
  • Source: Component where event originates (e.g., `Button`).
  • Event: Object describing the event (e.g., `ActionEvent`, `MouseEvent`).
  • Listener: Object implementing a listener interface (e.g., `ActionListener`), registered with the source, containing handler methods (e.g., `actionPerformed()`) that are called when the event occurs.

Unit 4: Input/Output, Databases, Networking, Collections

Q: What's the difference between Byte Streams and Character Streams in I/O?

  • Byte Streams (`InputStream`/`OutputStream`): Handle raw 8-bit binary data. Use for non-text files (images, etc.) or binary network data.
  • Character Streams (`Reader`/`Writer`): Handle 16-bit Unicode character data. Automatically handle character encoding/decoding. Use for text files.

Q: What do Buffered Streams do?

  • Improve I/O performance using an internal memory buffer.
  • Read/write data in larger chunks, reducing system calls.
  • Wrap around existing streams (e.g., `BufferedReader(new FileReader(...))`).
  • Recommended for most file/network I/O.

Q: What is JDBC?

  • Java Database Connectivity - standard Java API (`java.sql`, `javax.sql`) for database interaction.
  • Allows Java apps to connect to databases, send SQL queries, process results.
  • Provides database independence (using specific JDBC drivers).

Q: What are the basic steps for using JDBC?

  • 1. Load/Register Driver: (Often automatic now).
  • 2. Establish Connection: `DriverManager.getConnection(url, user, pass)` -> `Connection`.
  • 3. Create Statement: `connection.createStatement()` -> `Statement` (or `PreparedStatement`).
  • 4. Execute Query: `statement.executeQuery(sql)` -> `ResultSet` (for SELECT); `statement.executeUpdate(sql)` -> `int` (for INSERT/UPDATE/DELETE).
  • 5. Process ResultSet: Loop `while(resultSet.next())`, use `resultSet.getXXX()` methods.
  • 6. Close Resources: Close `ResultSet`, `Statement`, `Connection` (use `finally` or try-with-resources).

Q: What is Object Serialization?

  • Converting an object's state into a byte stream.
  • Used for persistence (saving to file), communication (sending over network, RMI), caching.
  • Class must `implement Serializable` (marker interface).
  • Fields not marked `transient` are serialized.
  • Reverse process is Deserialization (byte stream -> object).
  • Uses `ObjectOutputStream` / `ObjectInputStream`.

Q: What is a Socket?

  • An endpoint for network communication between two programs.
  • Uses protocols like TCP (reliable) or UDP (faster, connectionless).
  • Java classes (`java.net`):
  • `Socket`: Client-side endpoint (connects to server).
  • `ServerSocket`: Server-side endpoint (listens for client connections via `accept()`).
  • Provides `InputStream`/`OutputStream` for sending/receiving data.

Q: What is the Java Collections Framework?

  • Unified architecture (`java.util`) for storing and manipulating groups of objects.
  • Key parts: Interfaces (`List`, `Set`, `Map`), Implementations (`ArrayList`, `HashSet`, `HashMap`), Algorithms (`Collections.sort()`).
  • Benefits: Reduces effort, increases performance, promotes reuse, improves quality.

Q: What's the difference between a `List` and a `Set`?

  • `List`: Ordered sequence, allows duplicates, access by index (`ArrayList`, `LinkedList`).
  • `Set`: Unordered (usually), **no duplicates**, no index access (`HashSet`, `TreeSet`, `LinkedHashSet`).

Q: What is a `Map`?

  • Collection of unique key-value pairs.
  • Retrieve value based on its key (`get(key)`).
  • Keys must be unique.
  • Order depends on implementation (`HashMap`=unordered, `TreeMap`=sorted by key, `LinkedHashMap`=insertion order).

Q: What is an `Iterator`?

  • Standard interface to traverse elements in a collection sequentially.
  • Methods: `hasNext()` (check if more elements), `next()` (get next element), `remove()` (optional, safely remove last element retrieved).
  • Obtained via collection's `iterator()` method.
  • Enhanced `for` loop uses an Iterator implicitly for collections.

Q: What is RMI (Remote Method Invocation)?

  • Java mechanism allowing objects in one JVM to call methods on objects in another JVM (potentially on different machines).
  • Simplifies distributed programming in Java.
  • Uses remote interfaces, stubs (client proxy), and registry (naming service).
  • Handles network communication and data marshalling/unmarshalling.
